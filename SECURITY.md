# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка         |
| ------ | ----------------- |
| 0.3.x  | :white_check_mark: |
| 0.2.x  | :x:                |
| < 0.2  | :x:                |

## Сообщить о уязвимости

Если вы обнаружили уязвимость в безопасности, **НЕ создавайте публичный issue**.

Вместо этого:

1. Отправьте email на: security@pattmsc.online (или через GitHub Security Advisory)
2. Укажите:
   - Описание уязвимости
   - Шаги для воспроизведения
   - Потенциальное влияние
   - Предлагаемое исправление (если есть)

Мы ответим в течение 48 часов и предоставим план устранения уязвимости.

## Политика раскрытия

Мы следуем принципу ответственного раскрытия:

1. Вы сообщаете об уязвимости приватно
2. Мы подтверждаем получение в течение 48 часов
3. Мы работаем над исправлением
4. Мы выпускаем патч и обновление
5. После релиза мы публикуем информацию об уязвимости

## Рекомендации по безопасности

### Для разработчиков:
- Не коммитьте секреты, API ключи или пароли
- Используйте `.env` для чувствительных данных
- Регулярно обновляйте зависимости
- Проверяйте код на уязвимости перед merge

### Для пользователей:
- Используйте только официальные релизы
- Регулярно обновляйте до последней версии
- Не делитесь своими API ключами
- Используйте HTTPS для всех соединений

## Благодарности

Мы благодарим исследователей безопасности, которые ответственно сообщают об уязвимостях.
