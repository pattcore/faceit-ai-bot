# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.3.x   | :white_check_mark: |
| 0.2.x   | :x:                |
| < 0.2   | :x:                |

## Reporting a Vulnerability

Found a security issue? **Please DON'T create a public issue.**

Instead:

1. Send an email to: security@pattmsc.online (or use GitHub Security Advisory)
2. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)

I'll respond within 48 hours and work on a fix plan.

## Disclosure Policy

I follow responsible disclosure:

1. You report the vulnerability privately
2. I acknowledge receipt within 48 hours
3. I work on a fix
4. I release a patch and update
5. After release, I publish vulnerability details

## Security Recommendations

### For Developers

- Don't commit secrets, API keys or passwords
- Use `.env` for sensitive data
- Update dependencies regularly
- Check code for vulnerabilities before merging

### For Users

- Use only official releases
- Keep your installation up to date
- Don't share your API keys
- Use HTTPS for all connections

## Acknowledgements

I thank security researchers who report vulnerabilities responsibly.
